# /sowhat

Run a structured "So What" analysis. Your job is to push past surface observations and find the real business or customer consequence.

Use British English. No dashes; use commas, semicolons, or periods instead. Stay concrete. Prefer specific impacts over generic phrases like "drive value" or "improve experience". Keep language simple and assume an audience of busy senior stakeholders.

## Process

1. Ask the user to provide:
   - The problem, observation, or proposal to analyse.
   - Any relevant context or constraints.
   - Two to four focus areas (optional), such as: customer experience, operational risk, commercial impact, team focus.

2. Restate the core problem in one sentence.

3. Run the So What ladder five times in sequence, pushing progressively deeper with each step.

## So What Ladder Structure

For each level, state the consequence clearly, then ask: "So what does that mean?" Push the answer to the next level.

```
Level 1: [First order consequence of the problem or observation]
  So what: [What does this enable or risk?]

Level 2: [Second order consequence]
  So what: [Push closer to behaviour change or measurable outcomes]

Level 3: [Third order consequence]
  So what: [Link to business, risk, or strategy]

Level 4: [Fourth order consequence]
  So what: [What decision or action does this demand?]

Level 5: [Fifth order consequence]
  Final insight: [The sharpest, most actionable version of the original observation]
```

## Output Format

```markdown
## Restated problem
[One sentence capturing the essence of the problem in plain language.]

## So What Ladders

### [Theme 1]
- Level 1: [consequence]
  - Level 2: [consequence]
    - Level 3: [consequence]
      - Level 4: [consequence]
        - Level 5: [sharpest insight]

### [Theme 2]
...

## What this means
[Two to three sentences. The headline insight. What decision or action does the analysis demand?]

## Recommended next step
[One concrete action to take based on the analysis.]
```

## Rules

- Run the ladder at least five levels deep on the most important theme.
- Stay concrete at every level. If you find yourself writing "improve value," push further.
- Do not ask clarifying questions unless the input is genuinely ambiguous in a way that blocks progress.
- Never invent data or quotes. If evidence is missing, flag it as an assumption.
